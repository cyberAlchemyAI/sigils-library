# Concept Cards

Concept cards describe recurring implementation patterns observed in the repository.

Each concept card should answer:

- what the concept is,
- where it lives,
- what contract it owns,
- how to recognize it,
- which dependencies it may use,
- how it should be tested,
- which observability signals matter,
- where evidence exists.

Use `concept-card.md` as the starting template for new cards.
