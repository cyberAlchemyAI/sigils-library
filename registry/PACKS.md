# Packs

Packs are planned curated bundles of related sigils and spells.

A pack should exist only when a group of artifacts has a repeatable theme, installation story, and maintenance boundary. Until then, use the [Sigil Registry](SIGILS.md) and [Spell Registry](SPELLS.md) directly.

## Candidate Packs

- Repository knowledge harness: inventory, architecture pattern inventory, context builder, repository harness spell.
- Arcanum bootstrap: sigil runtime installer, observability setup, arcanum bootstrap spell.
- Ontology governance harness: inventory, ontology vault, context builder, optional architecture pattern inventory, ontology harness spell.
- Specification resilience: context builder, residuality spec, decision gate, inventory.
- Sigil authoring: sigil development, skill decomposer, skill transcriptor, spellcraft, signal observer, workflow reflect.
- Sigil runtime adapters: sigil runtime installer, sigil development, signal observer, workflow reflect.
- Sigil maintenance: signal observer, workflow reflect, sigil development, sigil maintenance loop spell.

## Entry Requirements

A pack should define:

- purpose,
- included sigils and spells,
- installation order,
- required local runtime folders,
- expected outputs,
- verification checks,
- maintenance owner or cadence.
